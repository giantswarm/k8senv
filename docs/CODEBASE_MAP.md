## System Overview

k8senv is a lightweight Kubernetes testing framework that manages on-demand kube-apiserver instances backed by kine (SQLite-based etcd shim) for efficient parallel test execution. Instances are created lazily, pooled for reuse, and isolated through namespaces. Release behavior is configurable via `ReleaseStrategy` (Restart, Clean, Purge, or None).

**ReleaseStrategy** (defined in `config.go`):
```go
type ReleaseStrategy int
const (
    ReleaseRestart ReleaseStrategy = iota  // Stop instance, DB restored from template on next acquire
    ReleaseClean                           // Delete non-system resources/namespaces via K8s API, keep running
    ReleaseNone                            // No cleanup, return as-is
    ReleasePurge                           // Direct SQLite DELETE WHERE id > baseline, keep running
)
```

**Release flow** (`instance.go`):
```
Release(token) → validate token (CAS on gen counter) → switch strategy:
  ReleaseNone    → ReleaseToPool (no cleanup)
  ReleaseClean   → cleanNamespacedResources() → cleanNamespaces() → ReleaseToPool or ReleaseFailed
  ReleasePurge   → purgeHandle.purge() → ReleaseToPool or ReleaseFailed
  ReleaseRestart → Stop() → ReleaseToPool or ReleaseFailed

guardReleasePanic: defer ensures pool slot returned even if cleanup panics
```

**cleanup.go** — Resource and namespace cleanup for `ReleaseClean`:
- `listUserNamespaces()` → fast path if empty
- `cleanNamespacedResources()` → `discoverDeletableGVRs()` (cached `ServerPreferredNamespacedResources`), concurrent deletion via errgroup (limit 10)
- `cleanNamespaces()` → convergence loop (up to 100 iterations, 10ms delay), `deleteAndFinalizeNamespace()` removes finalizers via Finalize subresource
- Resources MUST be deleted before namespaces (no cascade-delete in API-only mode)
- System namespace protection via `[4]string` fixed-size array (prevents `append`)

**purge.go** — Direct SQLite cleanup for `ReleasePurge`:
- `purgeHandle` opened eagerly at instance startup after `waitForSystemNamespaces`
- Captures baseline `MAX(id)` via `COALESCE(MAX(id), 0)` before any test writes
- Prepared `DELETE FROM kine WHERE id > ? AND name NOT IN/LIKE system patterns`
- WAL mode, `busy_timeout(5000)`, `synchronous(OFF)`, `MaxOpenConns(1)`
- Baseline query retries (3 attempts, 100ms exponential backoff) for `SQLITE_BUSY`
- Bypasses K8s admission chain, finalizers, RBAC entirely

**Key interactions**:
- `Manager` uses atomic state machine: `created → initializing → ready → shuttingDown`
- `Manager.Acquire()` returns `(*Instance, token uint64, error)` — token enables double-release detection
- `Manager` implements `InstanceReleaser` interface — wires Release callback from Instance back to Pool
- `Pool` creates instances on demand via factory; LIFO stack for cache locality
- `Instance` uses generation counter (atomic uint64, odd=acquired/even=free) — monotonic, never resets
- Client cache uses `atomic.Pointer[clientCache]` with CAS copy-on-write; `*[]GVR` distinguishes undiscovered (nil) from discovered-empty

### internal/kubestack — Process Stack

**Purpose**: Orchestrates coordinated kine + kube-apiserver startup/shutdown as a single unit.

| File | Purpose | Tokens |
|------|---------|--------|
| `stack.go` | Two-process lifecycle with errgroup, two-context pattern, StartWithRetry | 5052 |
| `doc.go` | Package documentation | 80 |

**Startup**: `errgroup.WithContext(readyCtx)` starts kine and apiserver in parallel. Apiserver tolerates kine not being ready via built-in etcd retry. Two-context pattern: `processCtx` (Background) for process lifetime, `readyCtx` (caller timeout) for startup.

**Shutdown**: Reverse order (apiserver first, then kine) — sequential. Each process gets full timeout (worst-case total = 2 × timeout). Uses `StopCloseAndNil` helper.

**Retry**: `StartWithRetry` validates once, retries `newValidated` + `Start` up to `maxRetries` times. `permanentStartErrors` list (permission, not-found, canceled, etc.) aborts immediately without retry.

**Port allocation**: `PortRegistry.AllocatePortPair()` for two distinct ports. Ports released in `Stop()`.

**Layer hierarchy**:
```
Stoppable interface { Stop(time.Duration) error; Close() }
    ^

    ^
├── kine.Process
└── apiserver.Process
```

**Process death detection**: `WaitReady` accepts `ProcessExited <-chan struct{}` — checked non-blocking before each readiness attempt for fast abort if process dies during startup.

**Stop sequence**: SIGTERM → grace period (5s, capped at half total budget) → SIGKILL via `time.AfterFunc` → drain reserve (2s). `expectSignalExit()` treats SIGTERM/SIGKILL exits as success. `StopCloseAndNil` guarantees Close + nil via defer regardless of Stop outcome.

**Two-channel design**: `waitDone` (buffered 1, consumed by Stop) vs `exited` (closed, N readers for WaitReady). Single `cmd.Wait` goroutine feeds both.

        Instance->>Pool: ReleaseToPool (stopped)

## Conventions

**Naming**:
- Interfaces in root package, implementations in `internal/`
- Functional options: `With[Setting](value) ManagerOption` (panic on invalid)
- Sentinel errors: `Err[Condition]` (e.g., `ErrShuttingDown`) — `const` via `sentinel.Error`
- Process types: `[Name].Process` with embedded `BaseProcess`
- Instance IDs: `inst-{index}-{8-hex-chars}` (random, not cryptographic)
- Release strategies: `Release[Behavior]` (e.g., `ReleaseRestart`, `ReleaseClean`, `ReleasePurge`, `ReleaseNone`)
- Type alias for strategy: auto-propagates new methods to public API; canary test catches unintentional additions

**Concurrency**:
- Prefer atomics for fast reads (`gen`, `started`, `state`, `clients`)
- Mutex only for complex state (Pool, PortRegistry, Instance.startMu)
- Generation counter (atomic uint64, monotonic) for ABA prevention on double-release
- Atomic counters for TOCTOU guards (inflight drain pattern)
- `sync.Once` for pool close, inflight done, stress config parse
- `errgroup.WithContext` for parallel operations (startup, cleanup, file prep)
- CAS copy-on-write for `clientCache` (up to 100 retries)
- Singleflight for discovery mapper refresh coalescing

**Error handling**:
- Sentinel errors for caller-facing conditions (`errors.Is()` compatible)
- `fmt.Errorf("context: %w", err)` wrapping throughout
- `errors.Join()` for collecting multiple errors (validation, shutdown, stop+close)
- Panics only for programmer errors (invalid options, double-release, nil factory)
- `guardReleasePanic` ensures pool slot returned even on cleanup panic

**Testing**:
- All tests are integration tests (`//go:build integration`) except root package unit tests
- External package pattern (`k8senv_test`) — only public API tested
- Tests fail (not skip) if binaries missing
- `t.Parallel()` for concurrent tests, `t.Cleanup()` for deferred teardown
- Nine test packages with separate singletons, strategy-specific configuration
- Two-phase execution: non-stress packages first, stress packages sequentially
- Shared helpers in `tests/internal/testutil/`, label-parameterized for reuse
- Canary tests (reflection-based field/method counts) catch uncovered schema changes
- Deterministic PRNG (`rand.PCG(workerID)`) for reproducible stress failures

**Logging**:
- `log/slog` throughout (structured logging)
- Package logger with `"component": "k8senv"` attribute
- Instance loggers add `"id": instanceID`
- Dual atomic pointer + CAS for lock-free logger access
- `K8SENV_LOG_LEVEL` env var for test verbosity

**Lint**:
- 55 linters enabled in `.golangci.yml` across 11 tiers
- Cyclomatic complexity: 15, Cognitive complexity: 20
- `//nolint:linter // justification` required for suppressions
- `nolintlint` enforces explanation + specific linter name
- `sloglint` enforces snake_case keys, no mixed args
- Test files exclude: gocyclo, gocognit, dupl, goconst, unparam, forcetypeassert

## Gotchas

1. **Manager is a process-level singleton** — `NewManager()` uses `sync.Mutex`. Options are ignored after the first call (logs warning with `configDiffs` listing conflicting fields). Cannot be reset without process restart (except `ResetForTesting()` in tests).

2. **ReleaseStrategy is manager-level, not per-call** — Strategy is configured once via `WithReleaseStrategy()` and applies to all instances. Cannot be changed per-release call.

3. **AcquireTimeout covers pool wait + instance startup** — with a bounded pool (default: 4), Acquire may block waiting for a free instance before creating or reusing one. Default 30s is sufficient for most cases; increase if pool contention is expected.

4. **Lazy initialization** — first `Acquire()` on each instance pays startup cost (~5-15s). Subsequent acquires may be instant (ReleaseClean/ReleasePurge/ReleaseNone) or pay startup again (ReleaseRestart).

5. **Token-based double-release detection** — `Acquire()` internally issues a token (generation counter). `Release()` validates via CAS. Double release from `instanceWrapper` returns `ErrDoubleRelease`; stale token at pool level panics (programmer error).

6. **Config() has TOCTOU window** — the released check in `Config()` is defensive, not a concurrency guarantee. Documented as intentional (caller error if racing with Release).

7. **Port allocation TOCTOU** — ports from kernel may be claimed before use. Mitigated by PortRegistry (prevents internal conflicts) and `StartWithRetry` (creates fresh Stack on port conflict, up to 5 retries).

8. **Two-context pattern in Stack** — `processCtx` (Background) keeps processes alive across boundaries; `readyCtx` (caller) only gates startup timeout. Must be different objects (validated).

9. **API-only mode** — no scheduler or controller-manager. Pods remain Pending. Controllers don't reconcile. Finalizers must be removed manually during cleanup (via Finalize subresource).

10. **Cached logger doesn't track slog.SetDefault()** — the cached default logger is set once via CAS. Call `SetLogger(nil)` to invalidate.

11. **CRD cache first run** — spins up a temporary kubestack, applies CRDs, waits for Established condition, copies the DB. Can take 30+ seconds. Subsequent runs use cached DB instantly.

12. **Stop timeout is per-process** — Stack stops apiserver then kine sequentially. Worst-case total = 2 × timeout.

13. **InstanceStopTimeout similarly per-process** — worst-case total instance stop = 2 × InstanceStopTimeout.

14. **CRD apply discovery retry** — `NoKindMatch` detection uses string matching because upstream doesn't implement `Unwrap`. Canary test (`TestApplyMissingKindErrSubstring`) catches format changes.

15. **Namespace cleanup requires finalizer removal** — API-only mode has no controller-manager, so namespace finalizers must be removed via Finalize subresource before deletion completes.

16. **Resource deletion before namespace deletion** — `ReleaseClean` must delete namespaced resources first because API-only mode has no cascade-delete on namespace deletion.

17. **Stress tests run in separate phase** — `Makefile.test.mk` runs stress packages sequentially after all other packages to avoid resource exhaustion.

18. **ReleasePurge bypasses K8s API entirely** — Uses baseline-ID `DELETE FROM kine WHERE id > ?` via `modernc.org/sqlite`. Finalizers, admission webhooks, RBAC all bypassed. Safe because instance is idle between Release and next Acquire, and `--watch-cache=false` ensures immediate consistency.

19. **Purge baseline captured at startup** — `MAX(id)` queried after `waitForSystemNamespaces` but before `started=true`. Ensures all system namespace rows are protected regardless of their IDs.

20. **System namespace protection is fixed-size array** — `[4]string` (not slice) prevents accidental `append`. `SystemNamespaceNames()` always returns a fresh copy.

## Navigation Guide

**To add a new functional option**: Edit `options.go` — add a `With[Name]` function. Add field to `internal/core/config.go` (`ManagerConfig`). Wire default in `defaults.go`. Update `config.go` (root). Update canary field counts in `config_test.go` and `export_test.go` (`ConfigSnapshot`).

**To add a new release strategy**: Add constant to `internal/core/config.go` (`ReleaseStrategy` enum). Update `IsValid()` and `String()`. Add case in `internal/core/instance.go:Release()`. Re-export in `strategy.go`. Update canary method count in `strategy_test.go`. Add test package under `tests/`.

**To add a new process type**: Create `internal/[name]/process.go`, embed `process.BaseProcess`, implement `Stoppable` (compile-time check). Wire into `kubestack/stack.go` for coordinated lifecycle.

**To modify instance startup**: Edit `internal/core/instance.go` (`Start` method) and `internal/kubestack/stack.go` (`Start` method).

**To change pool behavior**: Edit `internal/core/pool.go` — `Acquire`, `Release`, or `ReleaseFailed` methods.

**To add an integration test**: Add to appropriate file in `tests/` with `//go:build integration` tag. Use `sharedManager.Acquire()`, `testutil.UniqueName()`, `t.Parallel()`.

**To add a strategy-specific test**: Create new package under `tests/[name]/` with `main_test.go` (configure strategy in TestMain via `testutil.SetupAndRun`) and `[name]_test.go`. Share assertions via `testutil/release.go` with a label parameter.

**To add a CRD test**: Add to `tests/crd/crd_test.go`. Define CRD YAML in `testdata/`, add file creation in `setupSharedCRDDir()` in `testdata_test.go`.

**To add a pool exhaustion test**: Add to `tests/poolsize/poolsize_test.go`. Do NOT use `t.Parallel()` — these tests need exclusive pool access.

**To modify CRD caching**: Edit files in `internal/crdcache/` — `cache.go` for flow, `apply.go` for resource application, `hash.go` for cache key.

**To change kube-apiserver flags**: Edit `internal/apiserver/process.go` — modify `buildArgs()`.

**To change kine flags**: Edit `internal/kine/process.go` — modify arg construction in `Start()`.

**To add a linter**: Edit `.golangci.yml` — add to appropriate tier in `enable` list, configure in `settings`, add exclusions if needed.

**To add a stress test strategy**: Create `tests/stress[name]/` with `main_test.go` (configure strategy) and `stress_test.go` (call `testutil.StressWorker`). Add to Phase 2 exclusion list in `Makefile.test.mk`.
